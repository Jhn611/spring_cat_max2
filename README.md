# Весенний_код_40

Бот MAX для записи абитуриентов на мероприятия университета.

Реализовано:

- первое приветствие, дисклеймер хакатон-команды и согласие на обработку минимальных данных профиля;
- каталог мероприятий с датой, длительностью, форматом и свободными местами;
- подробная карточка мероприятия, выбор слота, подтверждение записи и код записи;
- отмена записи с учетом правила поздней отмены;
- уведомления по конкретной записи, которые пользователь может отключить;
- меню организатора: список закрепленных мероприятий, лимиты, список записей, поиск по коду, смена статусов;
- уведомления только активным участникам выбранного мероприятия.

## Быстрый старт

### Локально через Node

```bash
npm install
copy .env.example .env
npm run dev:service
```

Во втором терминале:

```bash
npm run dev
```

### Через Docker

```bash
copy .env.example .env
docker compose up --build
```

Или через npm-скрипт:

```bash
npm run docker:up
```

Остановить:

```bash
docker compose down
```

В `.env` нужно указать:

```env
MAX_BOT_TOKEN=полный_токен_бота
DATA_SERVICE_URL=http://localhost:3060
DATA_SERVICE_PORT=3060
DATABASE_PATH=data/springcat.sqlite
ADMIN_MAX_IDS=ваш_max_id
LEGAL_DOC_VERSION=hackathon-2026-05-14
```

В Docker `DATA_SERVICE_URL` можно оставить как в примере: `docker-compose.yml` сам переопределит его для контейнера бота на `http://data-service:3060`.

Чтобы узнать свой MAX ID, запустите бота и отправьте `/whoami`.

## Команды

- `/start` - главное меню
- `/events` - каталог мероприятий
- `/my` - мои записи
- `/org` - меню организатора
- `/find КОД` - найти запись по коду в меню организатора
- `/whoami` - показать MAX ID пользователя

## Настройка мероприятий

Демо-мероприятия находятся в `src/data-service/seed-events.ts`.

## Архитектура

Теперь бот и данные разделены:

- `src/bot` - модуль бота MAX. Он не работает с БД напрямую и обращается к `DATA_SERVICE_URL`.
- `src/data-service` - модуль сервиса данных: HTTP API, SQLite-слой и сиды мероприятий.
- `src/shared` - общий контракт: типы и доменные утилиты без зависимости от бота или БД.
- `data/springcat.sqlite` - локальная SQLite-БД.
- `Dockerfile` - собирает общий production-образ для обоих runtime-модулей.
- `docker-compose.yml` - запускает два контейнера: `bot` и `data-service`.

Сервис отдает мероприятия, свободные места, пользователей, записи, поиск по коду и обновление статусов. Бот занимается только диалогами MAX и вызывает сервис.

Такую структуру можно масштабировать вертикально и организационно: усиливать сервер сервиса данных отдельно от бота, заменить SQLite на PostgreSQL внутри `src/data-service`, вынести `src/data-service` в отдельный процесс или контейнер, а контракт в `src/shared` оставить стабильным.

## Логика по слоям

`src/bot`

- принимает события и callback-кнопки MAX;
- показывает каталог, карточки мероприятий, мои записи и меню организатора;
- не знает, как устроена БД;
- получает данные через `src/bot/data-service-client.ts`.

`src/data-service`

- поднимает HTTP API на `DATA_SERVICE_PORT`;
- владеет SQLite-файлом;
- создает таблицы пользователей, мероприятий и записей;
- сидит демо-мероприятия из `src/data-service/seed-events.ts`;
- считает свободные места и отдает записи/статусы боту.

`src/shared`

- хранит общие типы: пользователь, мероприятие, слот, запись, статус;
- хранит маленькие доменные функции вроде проверки активного статуса;
- не импортирует код бота, HTTP-сервера или БД.

## Docker-сервисы

`data-service`

- команда: `node dist/data-service/index.js`;
- порт наружу: `3060`;
- БД внутри контейнера: `/app/data/springcat.sqlite`;
- данные сохраняются в Docker volume `springcat-data`.

`bot`

- команда: `node dist/bot/index.js`;
- использует `MAX_BOT_TOKEN`;
- обращается к сервису данных по внутреннему адресу Docker: `http://data-service:3060`;
- наружные порты не открывает.

Для вертикального масштабирования можно переносить `data-service` на более мощную машину/контейнер с большим лимитом CPU/RAM. Для горизонтального масштабирования и нескольких экземпляров сервиса лучше заменить SQLite на PostgreSQL в `src/data-service/database.ts`, чтобы не делить один SQLite-файл между несколькими контейнерами.
